# 健康监测与智能分析平台项目整体架构与前端页面规划

## 一、规划依据

本项目采用 Tomcat 9.0 + JSP + Servlet 的 Java Web 课程设计结构。结合功能分析文档，系统应围绕用户健康主题组织页面，而不是按照开发人员分工或单一功能拆页面。

新的前端原则是：

- 页面按健康主题划分；
- 每个主题页面都是混合界面；
- 录入、历史、图表、预测、AI 建议和参考说明嵌入到同一个主题页中；
- `src` 包分工仍按课程要求保留，但不决定页面结构；
- 当前阶段先建立标准请求结构和页面跳转逻辑，数据库后续接入。

## 三、主题页面规划

| 页面 | 页面定位 | 页面内部混合模块 |
|---|---|---|
| `home.jsp` | 用户首页/健康总览 | 今日摘要、最近异常、快捷入口、各主题最新状态 |
| `body.jsp` | 体重体脂主题页 | 体重体脂录入、历史记录、BMI/体脂图表、体重趋势预测、AI 建议、相关文献 |
| `chronic.jsp` | 慢性病指标主题页 | 血压血糖血脂心率录入、历史记录、波动图表、异常提醒、AI 建议、参考范围/文献 |
| `lifestyle.jsp` | 运动睡眠主题页 | 运动睡眠录入、历史记录、趋势图表、运动睡眠建议、恢复状态分析、相关文献 |
| `report.jsp` | 综合健康报告页 | 多主题汇总、阶段性报告、跨指标分析、综合建议、预测摘要 |
| `admin.jsp` | 管理员页 | 用户列表、平台统计、BMI 分布、运动打卡统计、慢性病指标统计 |

`body.jsp`、`chronic.jsp`、`lifestyle.jsp` 都不是单纯录入页，而是围绕一个健康主题提供完整闭环。

## 四、可复用 JSP 区块

```text
WEB-INF/jsp/sections
├── inputPanel.jsp
├── historyPanel.jsp
├── chartPanel.jsp
├── predictionPanel.jsp
├── advicePanel.jsp
└── literaturePanel.jsp
```

复用方式：

- `body.jsp`：录入、历史、图表、预测、AI 建议、文献；
- `chronic.jsp`：录入、历史、图表、异常提醒、AI 建议、文献；
- `lifestyle.jsp`：录入、历史、图表、预测、AI 建议、文献；
- `report.jsp`：综合摘要、跨指标图表、综合建议、参考说明。

`sections` 通过 `request.setAttribute("topic", view)` 判断当前主题，并显示不同字段。

## 五、标准请求结构

页面访问：

```text
/page?view=home
/page?view=body
/page?view=chronic
/page?view=lifestyle
/page?view=report
/admin?action=home
```

主题接口：

```text
/auth?action=login
/auth?action=register
/auth?action=logout

/body?action=save
/body?action=queryHistory
/body?action=predict
/body?action=generateAdvice

/chronic?action=save
/chronic?action=queryHistory
/chronic?action=checkAbnormal
/chronic?action=generateAdvice

/lifestyle?action=saveSport
/lifestyle?action=saveSleep
/lifestyle?action=queryHistory
/lifestyle?action=predict
/lifestyle?action=generateAdvice

/report?action=generate

/admin?action=login
/admin?action=home
/admin?action=users
/admin?action=statistics
```

提交后回到对应主题页，例如：

```text
/body?action=save           -> /page?view=body&section=input
/body?action=queryHistory   -> /page?view=body&section=history
/body?action=predict        -> /page?view=body&section=prediction
/body?action=generateAdvice -> /page?view=body&section=advice
```

## 七、开发与验收重点

验收时可以说明：

- 项目已从单页面功能拆分改为健康主题混合页面；
- 每个主题页面都包含录入、历史、图表、建议、预测或异常提醒；
- 后端接口按主题动作组织，避免页面和接口一对一绑定；
- Java 包仍符合课程设计三人分工；
- 当前为模拟数据阶段，后续可继续接入 Dao 和数据库。
