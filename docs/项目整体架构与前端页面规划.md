# 健康监测与智能分析平台项目整体架构与前端页面规划

## 一、规划依据

本项目是 Java Web 课程设计项目，当前项目结构已经包含：

- `src/buxingchen`
- `src/chenqingyuan`
- `src/tanghaoli`
- `WebContent/static`
- `WebContent/WEB-INF/jsp`
- `WebContent/WEB-INF/web.xml`

结合《Java课程设计-功能分析文档》，系统定位为“健康监测与智能分析平台”。课程设计验收重点主要参考 `src` 和 `jsp`，因此项目架构应优先保证：

- `src` 中三个姓名拼音包职责清楚；
- `JSP` 页面数量适中，能覆盖主要功能；
- 页面、Servlet、业务处理之间关系明确；
- 基础功能完整，扩展功能可展示、可说明；
- 不追求过度复杂，重点体现 Java Web 项目的分层思想。

## 二、项目整体架构建议

本项目建议采用传统 Java Web 结构：

```text
WebContent
├── index.jsp
├── static
│   ├── css
│   │   └── style.css
│   ├── js
│   │   └── main.js
│   └── images
├── WEB-INF
│   ├── web.xml
│   ├── lib
│   └── jsp
│       ├── common
│       ├── user
│       ├── health
│       ├── analysis
│       └── admin
└── META-INF

src
├── buxingchen
├── chenqingyuan
└── tanghaoli
```

整体思路是：

- `WebContent` 负责前端页面、样式、脚本和 JSP 展示；
- `src` 负责 Java 后台代码，包括 Servlet、实体类、业务处理和数据库操作；
- `web.xml` 负责配置项目入口、字符编码、Servlet 映射等内容；
- JSP 页面负责表单提交和数据显示；
- Servlet 接收 JSP 请求，调用业务类处理数据，再转发回 JSP 页面。

## 三、WebContent 前端内容规划

### 1. WebContent 根目录

根目录建议保留系统入口页面：

```text
WebContent
└── index.jsp
```

`index.jsp` 作为系统首页，可以提供：

- 系统名称展示；
- 登录入口；
- 注册入口；
- 普通用户功能入口；
- 管理员入口。

建议首页不要写得过于复杂，重点是让验收老师能清楚看到系统主题和入口。

### 2. static 静态资源目录

```text
WebContent/static
├── css
│   └── style.css
├── js
│   └── main.js
└── images
```

各目录职责：

| 目录 | 作用 |
|---|---|
| `css` | 存放页面统一样式，例如布局、表格、按钮、表单样式 |
| `js` | 存放简单前端交互，例如表单校验、弹窗提示、图表数据展示 |
| `images` | 存放系统 Logo、健康主题图片等 |

课程设计中，前端不需要做成复杂框架，使用 JSP + CSS + 少量 JavaScript 即可。

### 3. WEB-INF/jsp 页面目录

建议将 JSP 页面放入 `WEB-INF/jsp` 下，避免用户直接通过地址访问内部页面，而是通过 Servlet 转发访问。

```text
WebContent/WEB-INF/jsp
├── common
│   ├── header.jsp
│   ├── menu.jsp
│   └── footer.jsp
├── user
│   ├── login.jsp
│   ├── register.jsp
│   ├── profile.jsp
│   └── userHome.jsp
├── health
│   ├── healthRecordAdd.jsp
│   ├── healthRecordList.jsp
│   ├── sportRecord.jsp
│   ├── sleepRecord.jsp
│   └── chronicRecord.jsp
├── analysis
│   ├── healthAnalysis.jsp
│   ├── healthReport.jsp
│   ├── prediction.jsp
│   └── aiAdvice.jsp
└── admin
    ├── adminLogin.jsp
    ├── adminHome.jsp
    ├── userManage.jsp
    └── statistics.jsp
```

## 四、JSP 页面功能分配

### 1. 公共页面

| JSP 页面 | 功能说明 |
|---|---|
| `common/header.jsp` | 页面顶部区域，显示系统名称、登录状态 |
| `common/menu.jsp` | 左侧或顶部菜单，提供用户端、管理员端功能入口 |
| `common/footer.jsp` | 页面底部版权或说明信息 |

公共页面可以通过 JSP include 引入，减少重复代码。

### 2. 用户账户页面

| JSP 页面 | 功能说明 | 对应后台 |
|---|---|---|
| `user/login.jsp` | 用户登录表单 | 登录 Servlet |
| `user/register.jsp` | 用户注册表单 | 注册 Servlet |
| `user/profile.jsp` | 健康档案维护 | 档案 Servlet |
| `user/userHome.jsp` | 用户登录后的首页 | 用户首页 Servlet |

该部分对应功能分析文档中的“用户账户管理功能”和“健康档案管理功能”。

### 3. 健康数据页面

| JSP 页面 | 功能说明 | 对应后台 |
|---|---|---|
| `health/healthRecordAdd.jsp` | 录入体重、体脂、血压、血糖等健康数据 | 健康记录 Servlet |
| `health/healthRecordList.jsp` | 查看历史健康数据 | 健康记录查询 Servlet |
| `health/sportRecord.jsp` | 录入步数、运动时间、运动打卡 | 运动记录 Servlet |
| `health/sleepRecord.jsp` | 录入睡眠时间并查看统计 | 睡眠记录 Servlet |
| `health/chronicRecord.jsp` | 录入血压、血糖、血脂、心率等慢性病辅助数据 | 慢性病监测 Servlet |

该部分是系统核心，应作为 JSP 验收重点之一。

### 4. 分析与智能功能页面

| JSP 页面 | 功能说明 | 对应后台 |
|---|---|---|
| `analysis/healthAnalysis.jsp` | 展示 BMI、基础代谢、热量消耗、异常提醒 | 健康分析 Servlet |
| `analysis/healthReport.jsp` | 展示近 30 天健康报告 | 报告 Servlet |
| `analysis/prediction.jsp` | 展示体重、体脂趋势预测 | 预测 Servlet |
| `analysis/aiAdvice.jsp` | 展示 AI 健康建议或模拟问答 | AI 建议 Servlet |

该部分可以作为项目亮点。若时间有限，AI 建议可以先采用规则模拟方式实现，例如根据 BMI、血压、睡眠时间输出固定建议，不一定必须真正接入大模型接口。

### 5. 管理员页面

| JSP 页面 | 功能说明 | 对应后台 |
|---|---|---|
| `admin/adminLogin.jsp` | 管理员登录 | 管理员登录 Servlet |
| `admin/adminHome.jsp` | 管理员首页 | 管理员首页 Servlet |
| `admin/userManage.jsp` | 查看和管理用户信息 | 用户管理 Servlet |
| `admin/statistics.jsp` | 查看平台统计数据 | 统计 Servlet |

管理员功能不需要过度复杂，重点体现多用户管理和平台统计。

## 五、src 三个姓名拼音包规划

当前 `src` 下已经建立三个包：

```text
src
├── buxingchen
├── chenqingyuan
└── tanghaoli
```

这符合“src 建包自己姓名汉语拼音”和“src 内建 3 个包”的要求。建议三个包按照功能模块分工，而不是每个包都重复写一套完整结构。

## 六、三个包的具体职责分配

根据课程设计分工表，姓名首字母与拼音包的对应关系如下：

| 分工标识 | 对应包名 | 负责模块 |
|---|---|---|
| `c` | `chenqingyuan` | 健康档案管理模块、个人数据录入模块、多用户管理模块、健康数据分析模块 |
| `b` | `buxingchen` | 前后端接口管理、健康数据历史查询模块、管理员统计模块 |
| `t` | `tanghaoli` | 智能辅助分析与建议模块、多维度健康数据预测、数据库管理 |

因此，后续 `src` 内三个包应按照该表进行开发，既满足“按姓名拼音建包”的要求，也能和课程设计分工表对应起来。

### 1. `chenqingyuan` 包：健康档案、数据录入、多用户管理与健康分析模块

- 用户登录；
- 用户退出；
- 用户健康档案；

## 七、前后端请求关系建议

建议使用如下请求方式：

```text
JSP 表单
  ↓
Servlet 接收请求
  ↓
Service 处理业务逻辑
  ↓
Dao 访问数据库
  ↓
Servlet 保存结果到 request/session
  ↓
转发到 JSP 展示
```

示例：

```text
healthRecordAdd.jsp
  ↓ 提交体重、体脂、血压、血糖
HealthRecordServlet
  ↓ 调用
HealthRecordService
  ↓ 调用
HealthRecordDao
  ↓ 返回结果
healthRecordList.jsp
```

这样写的好处是层次清楚，验收时能够说明每个类的作用。
