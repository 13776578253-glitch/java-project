# 按健康主题的混合界面与请求结构改造计划

## Summary

重新规划为“按健康主题组织页面”。每个主题页面内部都包含完整闭环：数据录入、历史记录、图表统计、基于历史的趋势预测、AI 健康建议、相关文献/参考说明。

不再单独设置“图表统计与智能分析”页面，因为统计、预测、AI 建议应该嵌入到具体健康主题中。

## Page Structure

保留四类主题页面：

| 页面 | 页面定位 | 页面内部混合模块 |
|---|---|---|
| `home.jsp` | 用户首页/健康总览 | 今日摘要、最近异常、快捷入口、各主题最新状态 |
| `body.jsp` | 体重体脂主题页 | 体重体脂录入、历史记录、BMI/体脂图表、体重趋势预测、AI 建议、相关文献 |
| `chronic.jsp` | 慢性病指标主题页 | 血压血糖血脂心率录入、历史记录、波动图表、异常提醒、AI 建议、参考范围/文献 |
| `lifestyle.jsp` | 运动睡眠主题页 | 运动和睡眠录入、历史记录、趋势图表、运动睡眠建议、恢复状态分析、相关文献 |
| `report.jsp` | 综合健康报告页 | 多主题汇总、阶段性报告、跨指标分析、综合建议、预测摘要 |
| `admin.jsp` | 管理员页 | 用户列表、平台统计、BMI 分布、运动打卡统计、慢性病指标统计 |

其中：

- `body.jsp` 不是单纯录入页，而是“体重体脂完整分析页”。
- `chronic.jsp` 不是单纯慢性病录入页，而是“慢性病指标监测分析页”。
- `lifestyle.jsp` 不是单纯运动睡眠页，而是“生活方式数据分析页”。
- `report.jsp` 只做跨主题汇总，不承接单项指标的主要分析功能。

## Standard Routes

页面路由按主题走：

```text
/page?view=home
/page?view=body
/page?view=chronic
/page?view=lifestyle
/page?view=report
/admin?action=home
```

模块接口按数据主题和动作走：

```text
/auth?action=login
/auth?action=register
/auth?action=logout

/body?action=save
/body?action=queryHistory
/body?action=predict
/body?action=generateAdvice

/chronic?action=save
/chronic?action=queryHistory
/chronic?action=checkAbnormal
/chronic?action=generateAdvice

/lifestyle?action=saveSport
/lifestyle?action=saveSleep
/lifestyle?action=queryHistory
/lifestyle?action=generateAdvice

/report?action=generate

/admin?action=login
/admin?action=users
/admin?action=statistics
```

提交后回到对应主题页：

```text
/body?action=save              → /page?view=body&section=input
/body?action=queryHistory      → /page?view=body&section=history
/body?action=predict           → /page?view=body&section=prediction
/body?action=generateAdvice    → /page?view=body&section=advice
```

其他主题同理。

## JSP Composition

调整 `WEB-INF/jsp` 为“主题页 + 可复用区块”：

```text
WEB-INF/jsp
├── common
│   ├── header.jsp
│   ├── footer.jsp
│   └── nav.jsp
├── auth
│   ├── login.jsp
│   └── register.jsp
├── user
│   ├── home.jsp
│   ├── body.jsp
│   ├── chronic.jsp
│   ├── lifestyle.jsp
│   └── report.jsp
├── sections
│   ├── inputPanel.jsp
│   ├── historyPanel.jsp
│   ├── chartPanel.jsp
│   ├── predictionPanel.jsp
│   ├── advicePanel.jsp
│   └── literaturePanel.jsp
└── admin
    └── admin.jsp
```

区块复用规则：

- `body.jsp` include：
  - `inputPanel.jsp`
  - `historyPanel.jsp`
  - `chartPanel.jsp`
  - `predictionPanel.jsp`
  - `advicePanel.jsp`
  - `literaturePanel.jsp`

- `chronic.jsp` include：
  - `inputPanel.jsp`
  - `historyPanel.jsp`
  - `chartPanel.jsp`
  - `advicePanel.jsp`
  - `literaturePanel.jsp`

- `lifestyle.jsp` include：
  - `inputPanel.jsp`
  - `historyPanel.jsp`
  - `chartPanel.jsp`
  - `predictionPanel.jsp`
  - `advicePanel.jsp`
  - `literaturePanel.jsp`

- `report.jsp` include：
  - 综合摘要区
  - 跨指标图表区
  - 阶段性报告区
  - 综合建议区

`sections` 中的 JSP 需要通过 request 属性判断当前主题，例如：

```text
topic=body
topic=chronic
topic=lifestyle
```

从而显示不同字段和标题。

## Servlet Plan

按主题拆 Servlet，而不是按页面或人员拆页面。

```text
buxingchen.FrontControllerServlet
```

负责：

- `/page`
- 根据 `view` 转发到主题 JSP
- 准备当前主题 `topic`、`section` 等页面参数

```text
chenqingyuan.UserServlet
```

负责：

- `/auth`
- 登录、注册、退出

```text
chenqingyuan.BodyServlet
```

负责：

- `/body`
- 体重、体脂、BMI、健康档案相关数据提交和查询入口

```text
chenqingyuan.ChronicServlet
```

负责：

- `/chronic`
- 血压、血糖、血脂、心率等慢性病指标录入和异常判断入口

```text
buxingchen.LifestyleServlet
```

负责：

- `/lifestyle`
- 运动、睡眠记录保存和历史查询入口

```text
tanghaoli.ReportServlet
```

负责：

- `/report`
- 综合健康报告生成

```text
tanghaoli.AdviceService / PredictionService
```

作为服务类被各主题 Servlet 调用，不单独对应一个主要页面。

例如：

```text
/body?action=predict
  ↓
BodyServlet
  ↓
PredictionService
  ↓
回到 body.jsp 的 prediction 区块
```

```text
/chronic?action=generateAdvice
  ↓
ChronicServlet
  ↓
AdviceService
  ↓
回到 chronic.jsp 的 advice 区块
```

## Navigation Rules

导航栏改为主题导航：

```text
健康总览     /page?view=home
体重体脂     /page?view=body
慢性病指标   /page?view=chronic
运动睡眠     /page?view=lifestyle
健康报告     /page?view=report
管理员       /admin?action=home
```

页面内部锚点或 section：

```text
/page?view=body&section=input
/page?view=body&section=history
/page?view=body&section=chart
/page?view=body&section=prediction
/page?view=body&section=advice
/page?view=body&section=literature
```

每个主题页都可以拥有相同的区块顺序：

```text
主题概览 → 数据录入 → 历史记录 → 图表趋势 → 趋势预测 → AI 建议 → 文献参考
```

## Assumptions

- 使用 Tomcat 9.0、JSP、Servlet、`javax.servlet.*`。
- 当前阶段先调整路由、页面结构和跳转逻辑，不接真实数据库。
- 图表先用静态区域或模拟数据展示，后续再接图表库。
- AI 建议和文献参考先使用规则文本和固定参考内容展示，后续可扩展为数据库或接口来源。
- 页面按健康主题组织；Java 包分工仍按课程设计要求保留。
